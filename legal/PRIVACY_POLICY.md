## 一、我们收集哪些信息

### 1.1 你主动提供的信息

| 信息类型 | 具体内容 | 何时收集 |
|----------|----------|----------|
| 账号信息 | 邮箱地址、经过哈希处理的密码 | 邮箱注册/登录时 |
| Apple 账号标识 | Apple 分配的唯一用户标识符（`apple_user_id`） | 使用「Sign in with Apple」登录时 |
| 用户创建内容 | 花园名称、植株名称与品种、成长记录文字备注、养护类型配置、养护提醒间隔设置、**植株标签（名称与颜色）**、**绿植资产信息（入手日期、入手价格）** | 你在应用内创建、编辑记录时 |
| 图片 | 你拍摄或选择的植物照片（成长记录照片、植株封面、识图照片） | 添加记录、设置封面、使用识图时（详见第 1.3、1.4 节） |
| 反馈信息 | 你提交的应用反馈或物种纠错内容（文字说明、可选附图）；提交应用反馈时附带的应用版本、设备平台与系统版本（用于定位问题） | 你在「设置 → 反馈与评价」或物种纠错入口提交时 |
| 个人资料 | 昵称（可选） | 修改个人资料时 |

### 1.2 自动生成的信息

| 信息类型 | 具体内容 | 用途 |
|----------|----------|------|
| 推送通知令牌 | 设备 APNs Token | 发送养护到期提醒 |
| 订阅标识符 | Apple App Store 原始交易 ID（`originalTransactionId`）、会员产品 ID | 验证和管理会员状态 |
| 会话凭证 | 加密存储在本机的 JWT access token / refresh token | 维持登录态、跨设备数据同步 |
| Apple 授权令牌 | 加密存储在服务端的 Apple refresh token（仅 Sign in with Apple 用户） | 账号注销时依 Apple 要求执行 token revoke |

### 1.3 图片的存储方式（重要说明）

你的成长记录照片与植株封面图**不会被上传到花迹自有服务器作为云端相册**。具体存储方式由你是否为付费会员决定：

- **免费用户**：图片**仅保存在你的设备本地**（App Documents 目录，受 iOS 整机 iCloud 备份保护），不上传至花迹服务器，也不逐张跨设备同步。换新设备或重装后，结构化数据会从花迹服务器恢复，但照片不会自动跟随。
- **付费会员**：图片保存在你设备的 **iCloud 容器**（iCloud Ubiquity Container），由 iOS 通过**你自己的 Apple ID 和 iCloud 配额**自动跨设备同步、换机自动恢复。**该同步过程由 Apple iCloud 完成，照片不经过花迹服务器。**
- 与图片关联的**结构化数据**（花园/植株/记录文字、标签、提醒配置等）会正常同步到花迹服务器；图片本体不在此同步范围内。

**仅在以下两种情形，图片会上传到花迹服务器**：① 你**主动提交反馈**时附带的图片；② 你**同意识图训练数据采集后**采集的识图照片（详见第 1.4 节）。这些图片与你的私人相册分开存储。

### 1.4 识图功能的信息处理

当你使用「识图（拍照认植物）」功能时：

- 你选择的照片会先在本机压缩，并在上传前**剥除 EXIF 元数据（含 GPS 位置等）进行隐私脱敏**。
- 脱敏后的照片会**临时传输给第三方植物识别服务**（该服务可能位于中国境外）以完成识别；识别完成后该临时图像即被清理，不作为云端相册长期保留。
- 识别记录（候选物种结果、图片哈希、你最终确认或选择的物种）会保存在花迹服务器，用于改进识别质量并完善植物库。
- **首次使用识图需经你明确同意**，同意内容包括：① 照片将传输给第三方识别服务以完成识别；② 花迹会保存你的识图照片与识别/确认结果，用于改进花迹自有的植物识别模型。**未同意则无法使用识图功能。** 同意行为会记录版本与时间戳以备审计。
- 用于改进模型的识图照片在入库前同样经 **EXIF 脱敏**后存储；你注销账号时，相关识图照片与识别记录会一并清理。
- 当识图结果用于为植物库补充新物种时，仅使用识别结果与你的确认信息，不公开你的账号身份。

### 1.5 我们不收集的信息

本应用**不收集**以下信息：
- 位置信息（不申请系统定位权限；照片 EXIF 中的 GPS 信息在上传识别前已被剥除）
- 通讯录
- 麦克风音频
- 面部识别数据
- 第三方广告追踪标识符（IDFA）

## 二、信息的用途

我们收集上述信息仅用于以下目的：

1. **提供账号服务**：注册、登录、维持登录状态、跨设备数据同步。
2. **保存和展示你的内容**：将你的花园、植株、成长记录、标签、资产等结构化数据存储在服务器，让你在多设备上访问和查看。
3. **识图识别**：将脱敏后的照片临时传输给第三方识别引擎完成植物识别，返回候选物种供你确认。
4. **改进识别能力**：在你同意后，使用识图照片与你的确认结果改进花迹自有的植物识别模型，并据此完善植物库。
5. **处理反馈**：回复你的反馈、修复问题，并根据你的物种纠错建议完善植物库内容。
6. **发送养护提醒**：根据你设置的养护间隔，通过 APNs 向你的设备发送推送通知。
7. **验证订阅状态**：向 Apple 服务器查询订阅收据，确认你的会员是否有效，并据此开放相应功能权限。
8. **账号安全与注销**：二次验证身份、执行冷静期删除流程，以及在你注销账号时依 Apple 要求吊销 Sign in with Apple 授权令牌。

**我们不会将你的个人信息用于商业广告投放、出售给第三方，或用于本政策未列明的目的。**

## 三、信息存储与保留

### 3.1 存储位置

- **结构化数据**（账号信息、花园/植株/记录文字、标签、资产信息、识别记录、订阅状态等）：存储在花迹自建服务器（国内区：腾讯云云服务器 + 云数据库）。
- **成长记录照片与植株封面**：免费用户仅存储在设备本地（App Documents）；付费会员存储在**你自己的 iCloud 容器**——这两种情形照片均**不经过花迹服务器**。
- **反馈附图、识图训练照片（仅在你同意后）**：经脱敏后存储在花迹对象存储服务（国内区：腾讯云 COS）。
- **本地 token**：仅存储在你的设备本地，不经过花迹服务器。

### 3.2 保留期限

| 数据类型 | 保留规则 |
|----------|----------|
| 正常账号数据 | 账号有效期间持续保留 |
| 申请注销后 | 进入 7 天冷静期；冷静期结束后对数据进行软删除和匿名化处理（花园/植株/记录/标签等与真实用户身份解除关联，邮箱等可识别信息从用户记录中清除） |
| 识图照片与识别记录（注销后） | 随账号注销流程一并清理 |
| 反馈数据 | 处理完成后按问题追溯需要保留合理期限 |
| 你手动删除的记录 | 关联的服务器图片（如反馈附图）标记为待清理，由定期任务处理；本机本地照片由设备本地清除；存入 iCloud 的照片由你的 iCloud 管理 |

## 四、信息共享与对外披露

我不会出售你的个人信息。仅在以下必要情形下，信息可能流转至第三方：

### 4.1 Apple

- **IAP 收据验证**：向 Apple App Store 服务器核查你的订阅收据，以确认订阅有效性。Apple 的隐私实践详见 [Apple 隐私政策](https://www.apple.com/legal/privacy/zh-cn/)。
- **App Store Server Notifications**：接收 Apple 主动推送的订阅状态变更通知（续费、到期、退款等）。
- **Sign in with Apple**：使用该登录方式时，Apple 会向花迹提供你的唯一标识符和（可选的）邮箱地址，相关处理遵循 Apple 的规定。
- **iCloud 图片同步**：付费会员的照片由 Apple iCloud 在你自己的 Apple ID 下同步，花迹不参与该过程、也无法读取你 iCloud 中的照片。

### 4.2 云基础设施提供商

花迹将服务器数据存储在**腾讯云（国内）**的服务器上。该服务商作为数据处理者，仅在花迹的指令下处理数据，不独立使用你的数据，其服务受其隐私协议约束。

### 4.3 第三方植物识别服务（识图）

使用识图功能时，经脱敏的照片会**临时传输**给第三方植物识别服务（该服务可能位于中国境外）用于识别，处理完毕后临时图像即被清除。这是识图功能的必要运作机制，**首次使用识图时会就此向你单独征得同意**；未同意则不会传输、识图功能不可用。识别服务商如有变更，将更新本政策。

### 4.4 法律要求

如法律法规、法院命令或政府机构强制要求，我们可能依法披露必要信息。

## 五、数据安全

我们采取以下措施保护你的数据安全：

- 账号密码在存储前经过单向哈希（bcrypt）处理，我们无法还原你的原始密码。
- 数据传输全程使用 HTTPS/TLS 加密。
- 服务端 JWT refresh token 使用 `jti` 唯一标识符，一旦吊销即立即失效，防止重放攻击。
- Apple refresh token（Sign in with Apple 用户）以加密形式存储在服务器，仅在账号注销时读取调用。
- 第三方 AI 识别服务的访问密钥以加密形式存储在服务器，不随代码或日志泄露。
- 识图照片在传输与入库前剥除 EXIF（含 GPS）元数据，做隐私脱敏。
- 服务器对象（反馈附图、植物库图鉴等）通过预签名 URL 方式访问，每次 URL 均有时效限制。

尽管如此，任何网络服务均无法保证 100% 的安全性。如遇数据安全事件，我将在合理时间内通过应用内通知或邮件告知受影响的用户。
